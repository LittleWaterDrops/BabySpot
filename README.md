# 25.03.10 구체화된 주제 공유 및 선정

### 1. 트렌드 분석을 통한 광고 자동생성 - 상화

- 소비 트렌드를 분석
- 대상을 한정적으로 좁히는게 유리할 것 같다 ex) 카페
- 특정 키워드의 검색량을 분석
- 카페 트렌드 키워드와 사용자가 선택한 이미지를 통해 생성형 AI를 활용한 광고 문구 & 짧은 광고영상을 제공

**피드백**

- 영상을 만들 수는 없음.
- 하지만 지금까지 생성형 ai로 이미지를 만들어 많이 활용하긴 했음.
- 키워드를 가져오고 빅데이터로 뭘 분석해야 할까 고민이 필요해 보임.
- 구글, 네이버를 통해 추출한 키워드는 구글, 네이버가 분석한 데이터임. 스스로 분석할 데이터가 필요함
- 빅데이터적인 부분이 여전히 모호해보임.
- SNS에서 크롤링을 가져온다고 할때, 자체적으로 문제가 있을 것

### 2. 코드 리뷰 자동화 서비스 - 병현, 한규

- Main : 사용자가 속해있는 집단 내의 코드 리뷰 데이터를 추출
- Sub : 오픈소스 데이터를 추출
- 많이 언급된 코드, 자주 수정된 코드를 중심으로 필터링
- 생성형 AI로 각 MR에 맞는 리뷰를 자동 생성

**피드백**

- 기업 내에서 사용이 불가능할 것.
- 코드 리뷰 서비스에 너무 의존하게 될 것.

### 3. 이동 약자 네비게이션 - 남주

- 이동에 불편함을 겪는 사람들이 안전하고 편리하게 이동할 수 있도록 돕는, 맞춤형 실시간 경로 안내 서비스
- 이동 유형을 선택하여 유형에 맞는 경로를 탐색

**피드백**

- 전부 api라는 것은, 우리가 모으는 데이터가 아니라는 것.
- 어느 고객이 어디로 갈지 모르는 상황에서 모든 교통정보 데이터를 가져오는게 맞을까?
- 데이터 필터링이 필요할 듯
  → 최근에는 막혀있지 않은 길인데, 과거에 막혀있는 데이터를 쌓아둘 필요가 없음
- 빅데이터를 사용한다는 것이랑 관련이 없어보임
  → 데이터를 쌓아서 무엇을 할 것인지가 필요함

### 4. 유아동반 베이비 스팟 추천 - 지승, 한규

- 유아 동반 가족이 편리하고 안전하게 이용할 수 있는 장소 추천
- GPS 및 빅데이터를 활용하여 사용자 맞춤형 장소 추천

**피드백**

- 개인화된 추천인지 잘 모르겠음.
- 아이와 관련된 리뷰가 있는지 크롤링한다는 부분은 좋음. 실제로 이 부분에서 어려움이 있기 때문에.
- 리뷰 크롤링만으로도 빅데이터에 적합하긴 함. 이후 서비스에 대한 고민이 필요할 듯
- 분산은 내부에서 일어나는 기술적인 측면이라, 분산을 사용한 추천 서비스라면 괜찮을 것

**피드백**

## 베이비 스팟 ✅

- 블로그 데이터 + 이미지 사용
- 당장 가야 할 지점에 대한 정보를 현재 내 위치 주변에서 알 수 있었으면 좋겠음
  - 어디에 갈 일이 있는데 갑작스럽게 해당 음식점을 찾아야 할 경우가 있기 때문에 더욱 필요성을 느낌
- 네이버 리뷰, 네이버 블로그를 사용
  → 내 아이와 비슷한 연령대인지 모호함 (사용할 수 있는 물품이 구비되어 있는가??)
  → OK존으로 가게를 추리고 이후 데이터를 수집하면 좋겠음.
  → 서울시만 하자
- 연령대 기준을 정해야 할 것 같음 (영아, 유아)
  - 웹 서칭을 할 때, 연령대를 기준으로 많이 검색을 하니 필요할 것이다.
  - 보통 연령대를 검색하는 이유는 물품을 사용가능한지 여부를 알기 위함인데, 이것은 아이들의 성장 속도에 따라 다 다르다.
    그러니 연령보다는 물품을 기준으로 판별하자
- 영유아의 구분은 필요해보임
  - 수유실, 놀이방 등등
  - 전자레인지 사용 유무 → 이유식
  - 세세하게 어느 부분까지 필요한지 알 수 없으니(너무 많아진다면 오히려 불편해지니) 나이의 구분도 필요해 보임

## 🚫 개발자 트렌드

- 기획과 정확히 똑같은 서비스가 이미 존재
- 분석할 만큼 많은 데이터가 필요한지 모르겠고, 답이 정해져 있는 느낌이 강함
- 규모별, 업종별로 나뉘지만 큰 변화는 없을 것
- 1년 단위로 키워드를 보여주는 사이트가 존재하지만 그 정도 텀으로는 서비스가 어려울 것. 빈번히 사용할 서비스가 필요해보임

## ✅ 인기 유튜버의 비결

- 제목은 어그로가 끌렸는데, 풀어가는 내용은 다큐인 느낌
- 인기 유튜버의 인기 추이를 파악할 수 있으니 좋아 보임. 다만 우리가 어떤 요소까지 접근이 가능할지 궁금함.
- 영상이 나오면 스크립트가 제공될 것임. 댓글처럼 스크립트에 접근하면 좀 더 납득 가능한 결과를 도출할 수 있지 않을까?
- 방향성 제시가 좋은 것 같음. 하지만 디테일하게 몇 분동안 해서 인기가 있어~ 는 어색해보임
- 해당 영상의 시간별 조회수를 따로 뽑을 수 있는지에 따라 달라질 것 같음
- 아이디어가 좋고 이후에 어떻게 풀어나갈지가 중요해보임
- 빅데이터에 어울려보임

## 🚫 제주도 랜덤 여행

- 단순 추천 서비스 아닌가 싶음
- 어느 방향으로 갈지 감이 안 옴
- 빅데이터와 연관성이 없어보임. 데이터만 넣고 랜덤으로 돌리는 것 같음
- 상위 지점에 점수를 매겨 판단한다고 했는데, 점수를 매긴 순간, 랜덤이 아님. 결국 같은 결과만 나오지 않을까 생각함. 단순 순위 매기기지 추천 서비스와도 거리가 멀어보임.
- 너무 전형적이라 우리가 고민할 부분이 적어보임.
- 여행에 재미요소를 더하거나, 개인화 되어있거나. 요소를 더 추가해야 할 듯
- 살면서 몇 번 안 가는데 개인화가 가능할지도 의문.
- 추천 퀄리티가 높지 않음. 이후에 검색해본 내역을 통해 데이터가 쌓이는 것
- 해당 주제를 많이 진행하긴 했음, 다만 지속적인 크롤링이 필요함

## 🚫 과일 정보 제공

- 가격이 다르다고 소비량에 큰 차이가 없다고 생각
- 이미 해당 서비스가 존재
- 정확히 어떤 데이터로 어떤 것을 제공한다는 것인지 명확하지 않아보임.
- 굳이 대상이 일반인일 필요가 있는지도 모르겠다. 빅데이터와의 연관성도 흐릿한 것 같음.
- 이 서비스의 목적성에 대한 고민이 더 필요해 보임
